# AI Интеграция для PCBuilderBot

## Обзор

Бот теперь поддерживает интеграцию с ИИ (Groq API) для умного подбора комплектующих ПК с автоматическим fallback на стандартную логику.

## Новые возможности

### 🤖 AI-усиленный подбор
- **Умное распределение бюджета** - ИИ анализирует рынок и оптимально распределяет средства
- **Объяснение выборов** - ИИ объясняет, почему выбраны конкретные компоненты
- **Адаптивность** - Учитывает актуальные цены и совместимость

### 🔄 Автоматический fallback
- **Надежность** - Если ИИ недоступен, автоматически используется проверенная логика
- **Бесшовность** - Пользователь не заметит переключения
- **Логирование** - Все случаи fallback логируются для анализа

## Установка и настройка

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка .env файла
Добавьте в `.env` файл:
```env
# Существующие настройки
TOKEN=your_telegram_bot_token

# Новые AI настройки
GROQ_API=your_groq_api_key
ENABLE_AI=true
AI_TIMEOUT=10
AI_MODEL=llama-3.1-8b-instant
AI_TEMPERATURE=0.3
FALLBACK_ON_ERROR=true
```

### 3. Получение Groq API ключа
1. Зарегистрируйтесь на [console.groq.com](https://console.groq.com)
2. Создайте API ключ в настройках
3. Добавьте ключ в `.env` файл

## Использование

### Запуск с AI
```bash
python main.py
```

### Отключение AI (fallback режим)
Установите в `.env`:
```env
ENABLE_AI=false
```

## Логика работы

1. **Попытка AI сборки** - Бот пытается использовать ИИ для распределения бюджета
2. **Валидация** - Проверяется корректность AI ответа
3. **Сборка** - Компоненты подбираются по AI квотам
4. **Fallback** - При любой ошибке используется стандартная логика
5. **Результат** - Пользователь получает сборку с индикатором использования AI

## Безопасность

- **Ключи в .env** - API ключи не хранятся в коде
- **Timeout** - Ограничение времени ответа ИИ
- **Валидация** - Проверка всех AI ответов
- **Fallback** - Гарантированная работа без ИИ

## Частые вопросы

### Q: Что если Groq API недоступен?
A: Бот автоматически переключится на стандартную логику сборки

### Q: Можно ли отключить AI?
A: Да, установите `ENABLE_AI=false` в `.env`

### Q: Какой модели используется?
A: По умолчанию `llama-3.1-8b-instant`, можно изменить в `.env`

### Q: Будет ли бот работать без API ключа?
A: Да, будет работать в стандартном режиме

## Поддержка

При проблемах с AI интеграцией:
1. Проверьте `.env` настройки
2. Убедитесь что Groq API ключ валиден
3. Проверьте логи на наличие ошибок
4. Установите `ENABLE_AI=false` для временного отключения
